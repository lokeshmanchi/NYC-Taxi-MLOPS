## Project Overview

MLOps pipeline for NYC Green Taxi fare prediction (Jan–May 2025, `data/`). Trains an XGBoost regression model wrapped in a scikit-learn Pipeline that handles feature engineering. Data loading uses Dask for scalability beyond RAM.

## Architecture

### Configuration — `src/config.py`

Central `pydantic_settings.BaseSettings` class (`AppConfig`) with env var overrides. Reads from `.env` file. Key groups:
- **Paths**: `model_path` (default `models/model.pkl`), `model_output_path` (default `models/model.pt`), `data_path`, `processed_data_path`
- **MLflow**: `mlflow_tracking_uri`, `experiment_name`
- **DDP**: `ddp_backend`, `rank`/`local_rank`/`world_size`, `batch_size`, `epochs`, `log_step_interval`, `grad_accumulation_steps`
- **Serving**: `max_batch_size` (default 256), `api_url`

### Feature Engineering — two-stage design

`src/features/core.py` is the canonical source for feature definitions and the transformer:

- **`BASE_FEATURE_COLS`** (8): raw/cleaned columns — `trip_distance`, `passenger_count`, `PULocationID`, `DOLocationID`, `pickup_hour`, `pickup_dayofweek`, `pickup_month`, `RatecodeID`
- **`FEATURE_COLS`** (11): `BASE_FEATURE_COLS` + derived `is_weekend`, `hour_sin`, `hour_cos`
- **`TARGET_COL`**: `"fare_amount"`
- **`TemporalFeatureEngineer`**: scikit-learn `TransformerMixin` that generates `is_weekend`, `hour_sin`, `hour_cos` on-the-fly

The derived features are **never stored** — always generated by `TemporalFeatureEngineer` to prevent training-serving skew.

`src/features/transform.py` provides `load_data()` (returns lazy Dask DataFrame, supports `s3://`/`gs://`), `filter_outliers()`, and `save_processed_data()` (materializes Dask graph to `data/processed/` Parquet + `_manifest.json`, then automatically calls `save_summary()` to generate `data/summary/hourly_summary.parquet`).

### Training pipeline — `src/training/train.py`

Builds a `sklearn.pipeline.Pipeline`:
```
[("features", TemporalFeatureEngineer()), ("model", XGBRegressor(...))]
```
Logged to MLflow via `mlflow.sklearn.log_model` (artifact path `model`, registered as **`nyc-taxi-regressor`**). Also exports the XGBoost step to ONNX + INT8-quantized ONNX (logged under `onnx/` artifact path). **No `joblib.dump` — the model artifact lives only in MLflow.**

### Model Serving — `src/serving/app.py`

Hybrid ONNX/sklearn serving via a lifespan context manager:
1. On startup, attempts to load an ONNX model first (if `MODEL_PATH` ends with `.onnx`)
2. Falls back to loading the sklearn pipeline via `fsspec.open(MODEL_PATH)` (supports cloud URIs)
3. Runs one warmup inference on startup
4. Endpoints: `GET /health`, `POST /predict` (single), `POST /predict_batch` (up to `max_batch_size`)
5. `GET /monitoring/drift` — generates and returns an Evidently HTML drift report

All `/predict` calls are logged to `data/predictions/` as date-partitioned Parquet for drift analysis.

**Important:** `MODEL_PATH` defaults to `models/model.pkl`. After retraining, either download the MLflow artifact to that path or set `MODEL_PATH` env var to the MLflow URI.

### Drift Monitoring — `src/monitoring/drift.py`

Uses [Evidently](https://docs.evidentlyai.com/) to compare live prediction inputs against the processed training distribution. The `GET /monitoring/drift` API endpoint calls `generate_drift_report()`:
- **Reference**: samples up to 5,000 rows from `data/processed/`
- **Current**: reads from `data/predictions/` (populated by `/predict` calls)
- **Output**: HTML report saved to `data/drift_report.html`

Requires at least one `/predict` call before the drift endpoint can generate a report.

### Edge Inference — `src/training/edge_run.py`

Standalone ONNX inference for edge/offline devices:
- Auto-selects hardware accelerator: **TensorRT (fp16) → CUDA → CPU** (provider priority at session load)
- Computes all 11 features inline via NumPy — no pandas/`TemporalFeatureEngineer` overhead
- LRU cache (1024 entries) on route signatures — repeated trips skip ONNX entirely
- SLA guard: if inference exceeds `LATENCY_BUDGET_MS` (10ms), falls back to a distance-based heuristic
- OTA hot-swap: background daemon thread polls the model file every 5 min; reloads in-place on mtime change with zero downtime
- Enforces minimum fare ($2.50)

`src/training/edge_benchmark.py` compares baseline (pandas + `TemporalFeatureEngineer`) vs optimized (NumPy + LRU cache) inference paths using real parquet data, reporting p50/p95/p99 latency, cache hit rate, and SLA compliance.

### Prefect Orchestration — two flows

| File | Flow | Purpose |
|------|------|---------|
| `src/pipelines/training_flow.py` | `training_pipeline` | Wraps `train.py` with retry logic (1 retry, 30s delay) |
| `src/serving/flow.py` | `etl_and_train_flow` | Runs Dask ETL then launches `torchrun` for DDP training |

### Distributed Training — `src/training/train_pytorch_ddp.py`

- `TabularNet`: MLP (128→64→1 with dropout)
- `ParquetStreamingDataset(IterableDataset)`: Streams rows from Parquet via `_manifest.json` — O(1) RAM
- DDP init: reads `torchrun` env vars, falls back NCCL→Gloo
- Gradient accumulation (configurable via `GRAD_ACCUMULATION_STEPS`), AMP mixed precision
- Saves per-epoch checkpoints (`models/checkpoint.pt`) compatible with fsspec (local/S3)
- Logs final model to MLflow registered as **`nyc-taxi-regressor-ddp`**; exports ONNX for edge

To run DDP locally:
```bash
python -m src.features.transform                           # Materialize + write _manifest.json
torchrun --standalone --nproc_per_node=4 src/training/train_pytorch_ddp.py
```

To run the Prefect ETL+DDP orchestration flow:
```bash
python -m src.serving.flow     # etl_and_train_flow: ETL → torchrun DDP
```

### Data flow
```
data/*.parquet
  → load_data() [Dask, lazy]
  → filter_outliers() + engineer_features() [Dask task graph]
  → .compute() [materializes to Pandas]
  → train_test_split
  → Pipeline.fit() [TemporalFeatureEngineer → XGBRegressor]
  → mlflow.sklearn.log_model → MLflow artifact store
  → API loads pipeline → pipeline.predict(BASE_FEATURE_COLS input)
  → _log_prediction() → data/predictions/ [date-partitioned Parquet]
```

## Key data schema notes

- Source: NYC TLC green taxi parquet, 21 columns, ~48k rows/month
- Pickup datetime: `lpep_pickup_datetime`
- `ehail_fee` is always null — not used
- `cbd_congestion_fee` present in 2025 data — not used as feature
- ~1836 rows/file have null `passenger_count`/`RatecodeID` — filled to 1 in `engineer_features()`
- Outlier filters: fare $2.50–$250, distance 0.1–60 mi, passengers 1–6
